# TaskTracker - Educational Task Management System

## Статус проекта: Активная разработка

Полнофункциональная веб-платформа для управления учебными задачами в образовательных учреждениях с системой ролевого доступа, уведомлений и аналитики.

## Функциональность

### Возможности студентов
- **Просмотр назначенных задач** с фильтрацией по статусу и приоритету
- **Изменение статуса выполнения** задач (Не начата/В процессе/Завершена)
- **Обмен задачами** с одногруппниками
- **Отслеживание дедлайнов** с системой уведомлений
- **Просмотр прогресса** через персональную статистику
- **Прикрепление фалов решений** как ответ на задачу
- **Написание комментариев** к задаче

### Возможности преподавателей
- **Создание и управление задачами** 
- **Назначение задач** отдельным студентам или целым группам
- **Мониторинг прогресса** студентов через систему метрик
- **Аналитика выполнения** с статистикой по группам
- **Управление учебными группами**
- **Оценка прикрепленных решений** сутдентами на задачу
- **Ответы на комментарии** к задачам
- **Создание отчётов** в формате excel файла

### Возможности администраторов
- **Полное управление пользователями** и их ролями
- **Создание и настройка учебных групп**
- **Назначение преподавателей** ответственным за группы
- **Системный мониторинг** и общая статистика платформы

## Технологии

### Backend
- **Java 17+** - основной язык программирования
- **Spring Boot 3.2.0** - основной фреймворк
- **Spring Security** - аутентификация и авторизация
- **Spring Data JPA** - работа с базой данных
- **JWT** - токены для безопасной аутентификации

### Frontend
- **Vue.js 3** - прогрессивный JavaScript фреймворк
- **Vue Router** - маршрутизация на стороне клиента
- **Axios** - HTTP клиент для взаимодействия с API

### База данных и кэширование
- **PostgreSQL 15** - реляционная база данных
- **Redis** - кэширование и управление сессиями
- **MongoDB** - хранение и работа с комментариями
- **Hibernate** - ORM для работы с данными
  

### Микросервисная архитектура
- **Apache Kafka** - асинхронная обработка событий
- **Docker** - контейнеризация сервисов

### Безопасность
- **JWT** с секретным ключом для аутентификации
- **BCrypt** для хеширования паролей
- **Ролевая модель доступа** (STUDENT/TEACHER/ADMIN)

## Установка и запуск

### Предварительные требования
- Java 17 или выше
- Maven 3.6+
- PostgreSQL 15+
- Redis 7+
- Docker и Docker Compose
- Python 3.9+ (для ML сервиса)

### Быстрый запуск через Docker

```bash
# Клонирование репозитория
git clone https://github.com/Danon003/todo.git
cd todo

# Запуск всей системы
docker-compose up --build
```

### Ручная установка

1. **Настройка базы данных**
```sql
CREATE DATABASE todo_db;
CREATE USER todo_user WITH PASSWORD 'todo_password';
GRANT ALL PRIVILEGES ON DATABASE todo_db TO todo_user;
```

2. **Запуск основных сервисов**
```bash
# Основное приложение
cd main-application
mvn clean package
java -jar target/todo-app-0.0.1-SNAPSHOT.jar

# ML сервис
cd ml-service
pip install -r requirements.txt
uvicorn app:app --host 0.0.0.0 --port 8000

# Frontend
cd frontend
npm install
npm run dev
```

3. **Конфигурация окружения**
Создайте `.env` файл:
```env
DB_NAME=todo_db
DB_USER=todo_user
DB_PASSWORD=todo_password
JWT_SECRET=your-jwt-secret-key
ML_SERVICE_URL=http://localhost:8000
REDIS_URL=redis://localhost:6379
KAFKA_BOOTSTRAP_SERVERS=localhost:9092
```

## База данных

### Основные сущности
```sql
persons (id, username, email, password, role, created_at)
tasks (id, title, description, deadline, priority, author_id, created_at)
task_assignments (task_id, user_id, status, assigned_at, assigned_by)
groups (id, name, description, teacher_id, created_at)
user_groups (user_id, group_id, created_at)
tags (id, name)
task_tags (task_id, tag_id)
```

### Связи и ограничения
- Один студент может состоять только в одной группе
- Задача может иметь multiple тегов через связующую таблицу
- Назначения задач отслеживают статус выполнения для каждого студента

## Безопасность

### Аутентификация и авторизация
- **JWT токены** с временем жизни для сессий
- **Ролевая модель** с разделением прав доступа
- **BCrypt хеширование** паролей пользователей

### Защита данных
- **Валидация входных данных** на всех уровнях приложения
- **SQL injection protection** через использование JPA
- **XSS protection** через экранирование вывода на фронтенде

### Безопасность API
- **CORS настройки** для контролируемого доступа к API
- **Rate limiting** для предотвращения злоупотреблений
- **Проверка прав доступа** на уровне методов и endpoints
