# Миграции баз данных

В этом каталоге содержатся миграции баз данных Flyway для приложения TaskTracker.

## Файлы миграций

- `V1__Initial_schema.sql` — создает начальную схему базы данных со всеми таблицами

## Как это работает

Flyway автоматически применяет миграции при запуске приложения. Миграции имеют версии и отслеживаются в таблице `flyway_schema_history`.

## Соглашение об именовании миграций

Миграции Flyway имеют следующий шаблон именования:
- `V{версия}__{описание}.sql`
- Пример: `V1__Initial_schema.sql`, `V2__Add_new_column.sql`

## Добавление новых миграций

1. Создайте новый файл SQL, следуя соглашению об именовании
2. Поместите его в папку `src/main/resources/db/migration/`
3. Flyway автоматически применит его при следующем запуске приложения

## Важные примечания

- Миграции применяются автоматически при запуске приложения
- Никогда не изменяйте существующие файлы миграции — вместо этого создавайте новые.
- Миграции отслеживаются в таблице `flyway_schema_history`
- Одни и те же миграции будут применяться как в локальной среде, так и в среде Docker
